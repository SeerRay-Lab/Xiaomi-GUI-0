Why RealMobile?

High benchmark scores do not reliably predict performance on real devices. Account states, permission dialogs, payment authentication, and risk-control mechanisms continually reshape the state distribution a GUI agent actually encounters — opening a persistent gap between offline scores and real-world usability.

RealMobile closes that gap by evaluating agents where they will be deployed: on physical phones, driving live commercial Chinese applications, on tasks drawn from real user traffic. Instead of a single pass/fail signal, each task is decomposed into ordered sub-goals that award partial credit, so an agent that completes most of a long-horizon task is measured fairly.

📱

Real Devices, Live Apps

Executed on physical phones against production applications — capturing permission dialogs, login walls, and risk control that emulators never surface.

🎯

Sub-Goal Scoring

Each task is broken into ordered sub-goals with partial credit (e.g. 0.33 → 0.66 → 1.0), verified with XPath and code rules over the UI-hierarchy XML plus PaddleOCR text.

🔗

Multi-App by Design

57% of tasks require coordinating two or more applications — cross-app search, information transfer, and comparison — reflecting how people actually use their phones.

🔍

Transparent & Reproducible

Every query, sub-goal rule, and scoring script is open. Recorded trajectories are re-scored deterministically, so results are auditable end-to-end.
